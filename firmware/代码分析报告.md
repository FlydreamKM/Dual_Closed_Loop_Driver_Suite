| 项目 | 内容 |
|------|------|
| **仓库地址** | https://github.com/FlydreamKM/Dual_Closed-LOOP_Driver |
| **MCU 平台** | STM32F103C8T6 (ARM Cortex-M3, 72 MHz / 64 KB Flash / 20 KB RAM) |

- 串级 PID 控制（位置环 → 速度环）
- 动态制动（能耗制动，过零自锁）
- 两套通信协议切换：VOFA+ (FireWater) 调参 或 二进制帧协议

## 三、当前通信协议说明

### 默认模式：VOFA-only（PROTOCOL_VOFA_ONLY = 1）

- **下行**：FireWater 文本命令，ASCII 文本行以 `\n` 结束
  - `M motor mode speed angle accel decel` — 设置目标
  - `P motor pid_type kp ki kd` / `PB pid_type kp ki kd` — PID 调参
  - `C motor cmd` — 控制指令（使能/失能/急停/回零/清除故障）
  - `B motor brake_pwm` — 动态制动
  - `V interval_ms` — 设置 JustFloat 输出频率
  - `S motor` — 请求状态（仅二进制模式有效）
- **上行**：VOFA JustFloat 波形帧，默认 200 Hz，10 通道实时数据
- **适用场景**：VOFA+ 上位机调参、波形观察、PID 整定

### 二进制模式（PROTOCOL_VOFA_ONLY = 0）

- **下行**：`0xAA 0x55` 帧头二进制协议，1 字节校验和（累加和低 8 位）
  - `0x01` SET_TARGET — 电机目标参数（19 bytes DATA）
  - `0x02` SET_PID / `0x07` SET_PID_BOTH — PID 参数设置
  - `0x03` CONTROL — 使能/失能/急停/回零/清除故障
  - `0x04` REQ_STATUS — 请求状态帧
  - `0x05` HEARTBEAT — 心跳保活
  - `0x06` SET_VOFA — 设置 JustFloat 频率（VOFA 模式专用）
- **上行**：
  - `0x81` STATUS — 周期性状态上报，默认 100 Hz，25 bytes DATA
  - `0x82` ACK — 通用应答
- **适用场景**：自定义上位机、高可靠通信

### 协议切换方式

修改 `Core/Inc/protocol.h` 第 32 行：
```c
#define PROTOCOL_VOFA_ONLY      0   /* 1=VOFA, 0=Binary */
```
重新编译烧录即可。

## 四、加减速度特性代码

文件：`Core/Src/controller.c` — `TrajectoryPlanner()` 函数

### 速度模式（MODE_SPEED）

- 目标速度为 `ctrl->target_speed`
- 每 1 ms 控制周期按 `accel` 加速度限制更新 `traj_speed`
- 代码逻辑：
  ```c
  float dv = target_sp - ctrl->traj_speed;
  float max_dv = ctrl->accel * CONTROL_DT_S;  // CONTROL_DT_S = 0.001
  if (dv > max_dv) dv = max_dv;
  else if (dv < -max_dv) dv = -max_dv;
  ctrl->traj_speed += dv;
  ```
- **注意**：速度模式只使用 `accel`，不使用 `decel`（减速特性由 PD 控制器自然处理）

### 位置模式（MODE_POSITION）

- 带梯形减速的判断逻辑：
  1. 计算当前位置误差 `pos_error = target_angle - traj_angle`
  2. 计算减速距离 `decel_dist = v^2 / (2 * decel)`
  3. 如果 `|pos_error| > decel_dist`：加速/巡航阶段
  4. 如果 `|pos_error| <= decel_dist`：按 decel 减速
  5. 速度变化用 accel 限幅
  6. 接近目标且速度低时锁住：`traj_speed=0, traj_angle=target_angle`

### 关键观察

- **加减速参数是独立存储的**：`ctrl->accel` 和 `ctrl->decel` 分别保存
- **SET_TARGET 命令重置 traj_speed 的位置**：文档说「每次 M 命令重置 traj_speed = 0 并清空 PID 积分」，但实际 `Controller_SetTarget()` 不重置。文档与实现不一致，需要确认意图

## 五、PID 控制代码

文件：`Core/Src/pid.c`

算法：位置式 PID（非增量式）

```c
float PID_Update(PID_t *pid, float setpoint, float actual) {
    float error = setpoint - actual;
    if (pid->first_run) { pid->prev_error = error; pid->first_run = 0; }
    pid->integral += error;
    float pre_output = pid->Kp * error + pid->Ki * pid->integral;
    // 抗积分饱和：输出超限时回退积分
    if (pre_output > pid->output_limit) {
        pid->integral -= error;
        pre_output = pid->output_limit;
    } else if (pre_output < -pid->output_limit) {
        pid->integral -= error;
        pre_output = -pid->output_limit;
    }
    // 积分限幅（最终防护）
    if (pid->integral > pid->integral_limit) pid->integral = pid->integral_limit;
    else if (pid->integral < -pid->integral_limit) pid->integral = -pid->integral_limit;
    // 重新计算 P+I + 微分项
    pre_output = pid->Kp * error + pid->Ki * pid->integral;
    float derivative = error - pid->prev_error;
    pid->prev_error = error;
    pid->output = pre_output + pid->Kd * derivative;
    // 输出限幅
    if (pid->output > pid->output_limit) pid->output = pid->output_limit;
    else if (pid->output < -pid->output_limit) pid->output = -pid->output_limit;
    return pid->output;
}
```

### 串级结构

```
位置模式：traj_angle → [位置环 PID] → 速度修正量 → 加到 traj_speed → [速度环 PID] → PWM
速度模式：traj_speed → [速度环 PID] → PWM
```

### 默认 PID 参数

| 环路 | Kp | Ki | Kd | 输出限幅 | 积分限幅 |
|------|----|----|----|---------|---------|
| 速度环 | 18.0 | 0.5 | 7.0 | ±1000 | ±500 |
| 位置环 | 40.0 | 0.0 | 1.0 | ±100 | ±50 |

## 六、切换到 Binary-Only 模式的修改建议

### 核心修改

1. **切换协议宏**：
   - `Core/Inc/protocol.h` 第 32 行：`#define PROTOCOL_VOFA_ONLY 0`

2. **SET_TARGET 添加积分重置**（文档 vs 实现不一致）：
   - 在 `App_ProcessCommand()` 的 `CMD_SET_TARGET` 分支中添加：
   ```c
   PID_Reset(&ctrl->speed_pid);
   PID_Reset(&ctrl->pos_pid);
   ctrl->traj_speed = 0.0f;
   ```

3. **清理 VOFA-only 条件编译排除的代码**（可选）：
   - `simple_atof()` 和 `Protocol_ParseTextLine()` 可删除
   - `vofa.c/h` 可在发布版中剔除

4. **心跳超时自动急停**：
   - 建议上位机断开后 500 ms 内无帧则自动进入 EMERGENCY

- **分支**：`binary-only`（已创建）
- **分析报告**：`代码分析报告.md`（本文件）
